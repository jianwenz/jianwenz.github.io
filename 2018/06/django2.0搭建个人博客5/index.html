Django2.0搭建个人博客(5)

Jun 28, 2018. Python, Django

设计博客首页和详情页

本章的blog主要包含了两个页面：

  • 首页
    展示所有博文的标题、预览等等。
  • 详情页
    详细展示某一篇博的内容，还可以包含评论、回复统计等等功能。

当然你还可以根据自己需要设计其他页面，实现思路都是类似的。

本章主要涉及到以下内容：

  • 选择前端框架
  • 配置静态文件
  • 模板继承
  • 设计首页和详情页view/url/template
  • 动态URL

5.1 下载Bootstrap4

Bootstrap是twitter基于HTML、CSS、JavaScript开发的简洁、直观、强悍的前端开发框架。
具有响应式布局等等一系列优点，能够大大简化web前端的工作量。
本文选择了Bootstrap4的blog模板，可以到这里下载本文使用的模板，或者也可以挑选一个其他主题或模板。

5.2 配置Django静态文件

回顾一下我们在前面关于Apache的配置，在apache我们已经配置了静态文件目录。

vim /etc/httpd/conf.d/django.conf    

LoadModule wsgi_module modules/mod_wsgi.so

<VirtualHost *:80>                 
    Servername jianwen.blog     
    Documentroot /var/www/mysite                        
    WSGIScriptAlias / /var/www/mysite/mysite/wsgi.py    
    ErrorLog logs/error_log                            
    Alias /static/ /var/www/mysite/blog/static/         #静态文件目录
    <Directory /var/www/mysite/blog/static/>
    Require all granted
    </Directory>
    <Directory /var/www/mysite/mysite>  
    Require all granted
    </Directory>
</VirtualHost>

创建静态文件的目录

mkdir /var/www/mysite/blog/static     #创建静态文件目录

拷入css/js
然后将下载的好css/js等静态文件copy到/mysite/blog/static/目录即可，下面一节会介绍如何在模板调用静态文件。

5.3 模板继承

创建base.html，并调用调用静态文件
我们创建一个base.html写入基础css/js、导航栏、页脚等代码，其他模板如index.html，post.html可以直接调用base.html，减少重复代码。

vim base.html

<!DOCTYPE html>
{% raw staticfiles %}
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1, shrink-to-fit=no">
    <meta name="description" content="">
    <meta name="author" content="">
    <title>Jianwen's Blog</title>
    <link rel="icon" href="{% static './source/icon/favicon.ico' %}">
    <!-- Bootstrap core CSS -->
    <link href="{% static './source/css/bootstrap.css' %}" rel="stylesheet">
  </head>

  <body>

....

{% block content %}
{% endblock %}

....

  </body>
    <!-- Bootstrap core JavaScript -->
    <script src="{% static './source/jquery/jquery.min.js' %}"></script>
    <script src="{% static './source/js/bootstrap.bundle.min.js' %}"></script>
</html>

{% raw staticfiles %}声明加载静态文件.
href="{% static './source/css/bootstrap.css' %}" 引入一个静态文件.
{% block content %}和{% endblock %}标记了模板继承的位置， content是block变量的名称，你也可以使用其他名称如{% block main %}。

view函数和urls配置
我们已经在第三章中完成了首页的view/url设置，有需要请返回查看相关配置。

修改index.html模板

{% extends "base.html" %}
{% block content %}
        <!-- Blog Entries Column -->
        <div class="col-md-8">

          <h1 class="my-4">Blogs Preview

       {% for blog in blog_list %}

5.5 博客详情页设计

对于每篇博文，我们希望能设置一个独立的URL，例如访问/post/1/是第一篇博文，/post/2/是第二篇博文。
最简单粗暴的解决方案就是给每篇博文设置一个view函数和urls，但是这明显不是很明智的方案。
Django给我们提供了一个方便的解决方法, 就是动态URL，即可以动态的为每篇博文分配一个url。

配置动态URL

vim mysite/urls.py

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', views.blog_index),
    path('post/<str:id>/', views.detail, name='detail'),   #增加此行，动态url
]

这样django将会根据id为每篇博文生成一个url,格式为/post/id。

编写view函数

vim view.py

#引入一个新的函数get_object_or_404
from django.shortcuts import render, get_object_or_404

#增加相应的view函数
def detail(request, id):
    blog = get_object_or_404(BlogsPost,id=str(id))
    return render(request, 'post.html', {'blog' : blog})

get_object_or_404的作用是如果博客id不存在，会报错404错误，id存在则会获取博客内容。

编写post.html模板

{% extends "base.html" %}
{% block content %}
        <!-- Post Content Column -->
        <div class="col-lg-9" style="margin:auto">
          <!-- Title -->
          <h1 class="mt-4">{{ blog.title }}</h1>
          <p class="lead">{% if blog.subtitle %}--{{ blog.subtitle }}{% endif %}</p>

          <!-- Date/Time -->
		  <hr>
          <p><span class="badge badge-dark">{{ blog.category }}</span> Posted on {{ blog.timestamp}}</p>

          <hr>

          <!-- Post Content -->
              {{ blog.body   }}

        </div>

{% endblock %}

同首页模板要继承base.html。

重启apache生效

service httpd restart

现在访问http://[server_ip]/post/[id]就看可以看到的详情页面了。
